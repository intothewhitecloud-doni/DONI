# Enterprise AI Decision Console

Enterprise AI Decision Console은 회사 내부 문서를 구조화해 의사결정에 필요한 운영 맥락을 도출하는 프로젝트입니다. 문서 안에 흩어진 업무 정보, 관계, 수치 근거를 연결해 조직이 같은 기준으로 현황을 이해하고 판단할 수 있게 만드는 것이 목표입니다.

## 배포 주소

배포 서버 주소 : https://doni-pi.vercel.app/

---

## 목표

기업의 의사결정은 보통 여러 문서, 표, 담당자 맥락, 업무 이력에 나뉘어 있습니다. 이 프로젝트는 그 정보를 하나의 운영 구조로 재구성해 다음 질문에 답할 수 있도록 돕습니다.

- 무엇을 관리해야 하는가?
- 어떤 업무흐름이 실제로 발생하고 있는가?
- 관리 대상과 업무흐름은 어떻게 연결되어 있는가?
- 어떤 지표가 판단 근거가 되는가?
- 지표와 관계를 바탕으로 어떤 인사이트를 만들 수 있는가?

## 핵심 개념

프로젝트의 앞단 구조는 다섯 가지 축을 중심으로 설계됩니다.

| 개념      | 설명                                                        |
| --------- | ----------------------------------------------------------- |
| 관리 대상 | 조직이 추적하고 판단해야 하는 주요 업무 객체                |
| 업무흐름  | 시간, 상태, 담당, 결과를 가진 업무 진행 단위                |
| 연결관계  | 관리 대상, 업무흐름, 지표, 인사이트 사이의 방향성 있는 관계 |
| 지표      | 문서와 업무흐름에서 계산되거나 추출되는 판단 기준           |
| 인사이트  | 연결관계와 지표를 기반으로 생성되는 의사결정 관점           |

의사결정 안건, 투표, 검증 이력은 이 구조 위에서 동작하는 후속 단계입니다. 현재 설계의 중심은 의사결정 전에 필요한 운영 구조를 정확하게 만드는 것입니다.

## 제품 흐름

1. 사용자가 업무 문서를 등록합니다.
2. 문서에서 관리 대상과 업무흐름 후보를 도출합니다.
3. 후보 간 연결관계를 분석해 운영 구조를 구성합니다.
4. 연결관계를 기반으로 지표를 계산하거나 매핑합니다.
5. 지표 변화와 연결 경로를 해석해 인사이트를 생성합니다.
6. 필요한 경우 인사이트를 의사결정 안건으로 전환합니다.

## 연결 구조

연결관계는 단순 텍스트가 아니라 방향성 있는 graph로 표현합니다. 각 edge는 출발점, 도착점, 관계 유형, 신뢰도, 근거, 관련 지표를 함께 가질 수 있습니다.

주요 edge 범주는 다음과 같습니다.

- 관리 대상 간 구조 관계
- 관리 대상과 업무흐름의 관계
- 업무흐름 간 순서 관계
- 업무흐름과 지표의 관계
- 지표와 인사이트의 관계

이 구조는 사용자가 어떤 근거에서 어떤 인사이트가 도출되었는지 역추적할 수 있게 하기 위한 기반입니다.

## 주요 화면 방향

- 데이터 보관함: 문서 추가, 미리보기, 다운로드, 분석 근거 확인
- 구조 검토: 추출된 후보 검토와 확정
- 관리 대상: 상위 관리 범주와 상세 연결 구조 확인
- 업무흐름: 업무 단계와 관련 관리 대상 확인
- 지표: 계산 기준, 근거, 관련 연결관계 확인
- 인사이트: 위험 신호, 원인, 권장 조치 확인
- 의사결정: 인사이트 기반 안건 생성과 투표 흐름

## 기술 스택

- Next.js
- React
- TypeScript
- Tailwind CSS
- Node.js 기반 테스트와 정책 검증

## 실행

```bash
npm install
npm run dev
```

개발 서버는 기본적으로 `http://localhost:3000`에서 실행됩니다.

## 검증

```bash
npm run typecheck
npm run test:policy
npm run build
npm run check
```

`npm run check`는 typecheck, 정책 테스트, production build를 순서대로 실행합니다.

## 설계 원칙

- 문서 분석 결과는 사람이 검토 가능한 구조로 표현합니다.
- 결과보다 근거와 연결 경로를 함께 보여줍니다.
- 관리 대상과 업무흐름을 먼저 명확히 분리합니다.
- 지표와 인사이트는 앞단 구조 위에서 생성합니다.
- 실제 의사결정 단계는 구조화된 운영 맥락을 기반으로 확장합니다.
